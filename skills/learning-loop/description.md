## Metadata
- **Name:** learning_loop
- **Version:** 1.0.0
- **Category:** intelligence

## Description

**THIS IS THE KEY SKILL — the soul of Hermes-Pi Factory Guardian.**

The learning loop demonstrates Hermes Agent's core differentiator: the ability to get smarter over time through experience, feedback, and pattern recognition. Unlike static monitoring systems that require manual reconfiguration, this skill continuously improves the factory guardian's understanding of what "normal" looks like for each machine, shift, and season.

### What It Does

1. **Records every event** — sensor readings, anomalies, alerts, operator feedback, machine state changes
2. **Finds repeating patterns** — "CNC Machine 3 always runs 5°C hotter during night shift" or "Conveyor startup always causes a vibration spike at 6 AM"
3. **Creates new Hermes skills from patterns** — Automatically generates skill YAML that can be registered with Hermes Agent
4. **Refines existing skills** — Updates anomaly thresholds, alert routing rules, and report content based on operator feedback
5. **Builds per-machine profiles** — Each machine develops a rich behavioral profile over weeks of monitoring
6. **Tracks improvement metrics** — Quantifies how much smarter the agent is getting

## Skill Generation

When a pattern is detected with sufficient confidence (>80%), the learning loop generates a new Hermes skill:

```yaml
# Auto-generated by learning_loop on 2025-06-15
# Pattern: Monday morning warmup sequence
# Confidence: 94.2%
# Events analyzed: 47

skill:
  name: monday_warmup_tolerance
  version: 1.0.0
  description: |
    On Monday mornings between 05:45–06:15, CNC machines 1 and 2
    and Conveyor 1 undergo a normal warmup sequence. During this period:
    - Temperature will rise from ambient to operating range over ~20 min
    - Vibration will spike to ~4.2g for ~30 seconds at startup
    - Current draw will peak at ~18A during initial motor engagement
    
    These readings should NOT trigger anomaly alerts.
    
  triggers:
    - day_of_week: monday
      time_range: [05:30, 06:30]
    
  actions:
    - suppress_anomaly:
        machines: [cnc_machine_1, cnc_machine_2, conveyor_1]
        sensor_types: [temperature, vibration, current]
        conditions:
          temperature: { max: 72.0 }
          vibration: { max: 5.0 }
          current: { max: 20.0 }
        duration_minutes: 25
    
  learning_behavior: |
    Monitor the warmup pattern each week. If timing drifts by more than
    15 minutes or values change by more than 20%, flag for review.
    If this pattern stops occurring for 2 consecutive weeks, 
    mark skill as inactive and alert operator.
```

## Feedback Integration

### From Operator Dismissals
```
Event: Operator dismisses alert as false positive
Learning action:
  1. Add reading to "normal" dataset for this machine+sensor+time
  2. Update baseline EMA
  3. If 3+ similar dismissals, create a tolerance rule
  4. Log: "Operator feedback: temp reading 71°C on CNC #1 is normal (3rd time)"
```

### From Operator Confirmations
```
Event: Operator confirms alert as real incident
Learning action:
  1. Record as confirmed incident in pattern database
  2. Tighten anomaly threshold by 5%
  3. Check if similar readings preceded past incidents (predictive pattern)
  4. Log: "Operator confirmed: CNC #3 stoppage was real. Threshold tightened."
```

### From Shift Reports
```
Event: Shift report shows trend (e.g., "CNC #1 running warm all shift")
Learning action:
  1. Analyze temperature trend over shift
  2. If trend is gradual, update baseline slowly (not a sudden anomaly)
  3. Create maintenance prediction if trend is concerning
  4. Log: "Gradual temperature increase detected on CNC #1. Updated baseline."
```

## Knowledge Export

All learned knowledge can be exported and imported:

```bash
# Export learned baselines and skills
python -m skills.learning_loop.learning_manager --export factory_knowledge.json

# Import on a new Pi (transfer learning!)
python -m skills.learning_loop.learning_manager --import factory_knowledge.json
```

This enables:
- **Fleet deployment:** Train on one Pi, deploy knowledge to all
- **Backup:** Never lose learned intelligence
- **Transfer learning:** Adapt knowledge from one factory to another
- **Compliance:** Audit trail of what the agent has learned
